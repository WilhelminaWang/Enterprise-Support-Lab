# 1. Scenario

A user can't join the domain.

![image-20260601144012332](./screenshots/image-20260601144012332.png)

# 2. Action taken

## 2.1 Check the line cable

- Check the Ethernet cable not Wi-Fi

  - check the icon of Ethernet cable
  - or cmd -> ipconfig to check
  - or check the adaptor

<img src="./screenshots/image-20260601204700216.png" alt="image-20260601204700216" style="zoom:50%;" />

<img src="./screenshots/image-20260601204401656.png" alt="image-20260601204401656" style="zoom:50%;" />

- Ping the server to check if the server is down.

  ```
  Ping mina.local
  ```

  It can connect to the server but the problem still exist, so it maybe the problem of the client side.

  <img src="./screenshots/image-20260601144215354.png" alt="image-20260601144215354" style="zoom:50%;" />



## 2.2 Delete the hostname from the AD

- Check the hostname of the client side

  ![image-20260601144826013](./screenshots/image-20260601144826013.png)
